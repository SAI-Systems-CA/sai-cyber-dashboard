Framework Spotlight - MITRE ATLAS v5.4 & OWASP Agentic Top 10 (2026)

MITRE ATLAS v5.4.0 (February 2026 update) now ships with 16 tactics, 84 techniques, 56 sub-techniques, 32 mitigations, and 42 real-world case studies. The 2026 release shifts focus from model-centric attacks to execution-layer exposure: autonomous workflow chaining, delegated-authority persistence, and API-level orchestration risk. New agentic case studies added this cycle include SesameOp (AML.CS0042), iProov deepfake-based KYC bypass, and MCP server compromise scenarios. All data is published in STIX 2.1 for ingestion by SIEM/TIP pipelines.

High-frequency ATLAS technique IDs to tag in detection logic: AML.T0040 (ML Model Inference API Access), AML.T0051 (LLM Prompt Injection), AML.T0054 (LLM Jailbreak), and the newer AML.T006x series covering agent hijack, tool abuse, and memory poisoning.

OWASP Top 10 for Agentic Applications 2026 (ASI) — finalized December 2025, peer-reviewed by 100+ contributors — is now the benchmark for autonomous-AI risk. It extends (does not replace) the OWASP Top 10 for LLMs. Two core principles anchor the list: Least-Agency (grant agents only the autonomy required) and Strong Observability (comprehensive logging of what agents do, why, and which tools they invoke).

ASI01 Agent Goal HijackRedirecting plans via injected instructions or poisoned content.
ASI02 Tool Misuse & ExploitationUnsafe chaining, ambiguous instructions, manipulated tool outputs.
ASI03 Identity & Privilege AbuseExploiting delegated trust and inherited credentials.
ASI04 Agentic Supply Chain VulnerabilitiesTampered third-party agents, tools, plugins, registries.
ASI05 Unexpected Code ExecutionAgent-generated or agent-invoked code becoming RCE.
ASI06 Memory & Context PoisoningCorrupting memory, embeddings, RAG stores to bias future reasoning.
ASI07 Insecure Inter-Agent CommunicationSpoofing, interception, weak A2A authentication.
ASI08 Cascading FailuresFalse signals propagating through automated pipelines.
ASI09 Human-Agent Trust ExploitationConfident explanations misleading operators to approve harm.
ASI10 Rogue AgentsAgents drifting beyond intended scope.

Cross-walk guidance: the NSA/CISA/FBI CSI Deploying AI Systems Securely joint guidance (April 2024) already maps directly to ATLAS v4.0 and the OWASP LLM Top 10. As of today, that cross-walk should be re-anchored to ATLAS v5.4 and ASI 2026 — particularly for control families Protect model weights, Actively monitor model behavior, and Secure exposed APIs.

LLM Supply Chain Research Spotlight - Huang et al. (2024) Re-applied

Today's briefing is dominated by Stage 1-2 failures under the Huang taxonomy, with a new Stage 4 (Operation) escalation. The vLLM chain (Items 3, 4, 10) spans three textbook SR2 patterns at the Platform and Model-Repository layers: RCE via multimodal parser chain, model-load code injection that bypasses trust_remote_code, and batch-runner SSRF. The Antigravity find_by_name flag-injection (Item 2) and PhantomRaven npm hallucination-squatting (Item 8) are SR2 at the Toolchain and Third-Party-Libraries components. Postmark-impersonation MCP (Item 9) is classic PR1 Data Leakage via Trojan tooling.

New pattern - Stage 4 operational weaponization: The Mexican-government Claude Code + GPT-4.1 campaign (Item 1) is the clearest real-world instance yet of SR1 Hacker Use at state-scale - commercial LLMs as the primary execution engine for reconnaissance, RCE generation, and exfiltration orchestration. This complements, rather than replaces, the Item 8 supply-chain story: attackers are poisoning the inputs developers trust (hallucinated package names) while also using the same models offensively against targets.

Mitigation mapping: Huang's Stage 1-2 defenses - SBOM, signed model manifests, CI/CD hardening, load-time sandboxing - now need to be paired with Stage 4 controls: agent-platform egress telemetry (ATLAS AML.T0096), per-agent identity, and write-path audit on RAG and system-prompt stores. A single integrated control set satisfies NIST AI RMF Manage, NIST 800-53 SA-12/SR-3/SI-4/AU-12, ISO 27001 A.14.1.2/A.14.2.7, and ITSG-33 SA-12/SI-4.
